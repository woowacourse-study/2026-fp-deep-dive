## 1. 액션·계산·데이터로 분류한 결과
### 액션
1. 크게 4가지 정도의 큰 액션으로 분류할 수 있었고, 핵심 로직으로서 service.js에 모아두었다.
- registerMember(id, name, point) 
- makeReservation(roomId, date, startHour, duration, attendees)
- cancelReservation(reservationId, hoursUntilStart)
- printMemberSummary()

### 계산
1. 생각보다 범용적인 유틸 함수로 많이 분리하였다.
2. 하나의 연산만 하고 다른 함수를 호출하지 않아 역할 분리가 명확하게 되었다.
3. 암묵적 입력과 암묵적 출력을 고려하니 명확하게 계산인지 아닌지 판단할 수 있었다.
4. 카피-온-라이트 기법으로 원본을 변형하지 않고 복제본을 만들어 return 하는 계산 함수를 적극적으로 사용하였다.

### 데이터
1. 기존 상수 데이터 말고도 추가적으로 필요하다고 판단하는 데이터를 만들어서 유지보수가 쉽도록 하였다.
- GRADE = ["normal", "honor", "master"];
- STATUS = { CONFIRMED: "confirmed", CANCELLED: "canclled" };
- PENALTY_PERCENT_UNTIL_ONE_DAY = 20;
